# 🛡 Nakrutka Guard Bot

Telegram-бот для защиты групп и каналов от накрутки подписчиков/участников.

[ВИДЕО с кратким обзором функций бота на Youtube](https://www.youtube.com/watch?v=90jEHVfew1s)

Если вам надо почистить группу или канал от УЖЕ НАЛИТЫХ вам ботов, то используйте [ВОТ ЭТОТ ПРОЕКТ](https://github.com/dvygolov/TG.GroupBotsCleaner).

## Особенности

- ⚡ **Высокая производительность** - асинхронная обработка через aiogram 3.x, in-memory счётчик вступлений
- 🎯 **Умная детекция** - скользящее окно для анализа вступлений
- 🔄 **Автоматическая защита** - включается при превышении порога, отключается при нормализации
- 🎯 **Скоринг пользователей** - ML-подобная оценка подозрительности по 7 параметрам (язык, аватары, username, качество username, имя, ID, premium)
- 🤖 **Капча для групп** - математические задачи с текстовым вводом ответа (опционально, 60 с таймаут)
- 👋 **Приветствия и правила** - кастомные HTML-сообщения, автоудаление через 3 мин
- 🚫 **Стоп-слова** - удаление сообщений с оскорблениями/спамом
- 🧹 **Чистка системных сообщений** - join/leave/kick скрываются мгновенно
- 👑 **Защита Premium** - пользователи с Telegram Premium могут спокойно подписываться на канал/в группу давже когда идёт атака.
- 📊 **Детальное логирование** - файловые логи по чатам и датам
- 🧾 **История автокорректировок скоринга** - видно когда и почему менялись веса/порог
- 📰 **Ежедневный дайджест админу** - сводка по всем чатам за 24 часа (если были события)
- ⚙️ **Гибкая настройка** - управление через inline-клавиатуру

## Как работает

### Режимы защиты

1. **Обычный режим**: 
   - Бот считает вступления в скользящем окне времени
   - Опционально: показывает капчу новым участникам (только для групп)

2. **Детекция атаки**: 
   - При превышении порога (например, 10 вступлений/мин) активируется защита

3. **Режим защиты**: 
   - Все вступающие кикаются автоматически (кроме Premium если настроено)
   - Капча не показывается

4. **Автоотключение**: 
   - Когда поток вступлений возвращается в норму, защита отключается
   - Капча снова включается (если была настроена)

### Скоринг пользователей

**Умная фильтрация** - анализирует профиль пользователя и кикает подозрительных автоматически. Работает для групп и каналов.

#### Как работает:
1. **Работает только в обычном режиме** (не в атаке)
2. Проверяется каждый новый участник
3. Вычисляется risk score (0-100) по формуле:

**Параметры оценки:**
- 🌍 **Язык** (до +30): редкие для вашего чата языки = подозрительно
- 🖼 **Аватары** (до +15): 0 фото = +15, 1 фото = +5, 2+ фото = норм
- 👤 **Наличие username** (+5): отсутствие username = подозрительно
- 🎲 **Качество username** (+10): рандомные username типа `user12345`, `qwerty777` = подозрительно
- ✍️ **Имя** (+10): нет латиницы/кириллицы = фейк
- 🌏 **Арабские/CJK символы** (+25): наличие арабских или китайских символов в имени = высокий риск
- 🆔 **Новизна ID** (до +20): слишком новый аккаунт = риск
- 👑 **Premium** (-20): бонус за Telegram Premium

**Действия:**
- Если score > порог (по умолчанию 50) → **кик без капчи**
- Если score ≤ порог → пропуск к капче (если включена) или в чат

**Обучение:** Юзеры, прошедшие капчу или скоринг, добавляются в статистику для улучшения точности (последние 7 дней).

### Капча (только для групп)

- **Математические задачи** с эмодзи (только цифры 0‑9)
- **60 секунд** на ответ + автоматический таймер
- **Текстовый ввод** - пользователь пишет ответ цифрами в чат
- **Автоудаление** ответов и всех сообщений от юзера, пока не пройдёт капчу
- **Одна попытка** - неправильный ответ или таймаут = автобан
- **Защита от ботов** - невозможен брутфорс кнопками, требуется парсинг задачи
- **Приветствие** показывается сразу после успешной капчи (если настроено)

## Установка (Linux)

### 1. Клонировать репозиторий
```bash
git clone <repo_url>
cd TG.NakrutkaGuard
```

### 2. Собрать окружение
```bash
chmod +x build.sh
./build.sh
```

Скрипт создаст виртуальное окружение `.venv` и установит зависимости.

### 3. Настроить .env
```bash
cp .env.example .env
nano .env  # или vim/любой редактор
```

Отредактировать `.env`:
```env
BOT_TOKEN=ваш_токен_от_BotFather
ADMIN_IDS=123456789,987654321
```

### 4. Запустить бота

**Вручную (foreground):**
```bash
.venv/bin/python -m bot.main
```

**В фоне через nohup:**
```bash
chmod +x start.sh
./start.sh
```

**Остановить:**
```bash
chmod +x stop.sh
./stop.sh
```

**Как systemd service (автозапуск при перезагрузке):**
```bash
chmod +x install-service.sh
sudo ./install-service.sh
```

Управление службой:
```bash
# Статус
sudo systemctl status nakrutkaguard

# Остановить
sudo systemctl stop nakrutkaguard

# Запустить
sudo systemctl start nakrutkaguard

# Перезапустить
sudo systemctl restart nakrutkaguard

# Логи
tail -f nakrutkaguard.service.log
```

**Удалить службу:**
```bash
sudo ./install-service.sh -u
```

### 5. Обновление (production)
```bash
chmod +x update.sh
./update.sh
```
Скрипт делает `git pull`, пересобирает зависимости через `build.sh` (без cache) и рестартует systemd сервис `nakrutkaguard`.

## Настройка бота

### Получение токена
1. Найдите [@BotFather](https://t.me/BotFather) в Telegram
2. Отправьте `/newbot` и следуйте инструкциям
3. Скопируйте токен в `.env`

### Добавление админов
В `.env` укажите ID админов через запятую:
```env
ADMIN_IDS=123456789,987654321
```

Узнать свой ID: [@userinfobot](https://t.me/userinfobot)

## Использование

### 1. Добавить чат под защиту
- Отправьте боту `/start`
- Нажмите "➕ Добавить чат"
- Перешлите сообщение из нужного чата или отправьте его ID

### 2. Добавить бота в чат/канал
⚠️ **Важно**: Бот должен быть **администратором** с правами:
- ✅ Удаление пользователей (Ban users)
- ✅ Приглашение пользователей (Invite users)

### 3. Настроить параметры
- **Порог** - количество вступлений для активации защиты (по умолчанию: 10)
- **Временное окно** - период в секундах для подсчёта (по умолчанию: 60)
- **Premium защита** - не кикать Telegram Premium пользователей (по умолчанию: Да)
- **🎯 Скоринг** - включить/выключить умную фильтрацию (для групп и каналов)
- **Капча** - отдельный переключатель для групп (в каналах недоступно)
- **Приветствие/Правила** - через кнопки «👋 Приветствие» и «📜 Правила» можно задать HTML-тексты с плейсхолдером `{username}`
- **Стоп-слова** - кнопка «🚫 Стоп-слова», список задаётся строками или через запятую; чтобы сбросить, отправьте `off`

### Глобальные команды дайджеста
- `/digest` — показать статус дайджеста и текущее время рассылки
- `/digest on` / `/digest off` — включить или выключить ежедневный дайджест
- `/digest_time HH:MM` — установить время ежедневной рассылки (локальное время сервера)
- `/digest_now` — отправить дайджест сразу (ручной запуск за последние 24 часа)

### Автоудаление контента
- Сообщения join/left/kick скрываются мгновенно
- Сообщения от pending-юзеров (не прошедших капчу) удаляются
- Любой текст/подпись, содержащий стоп-слово, удаляется без предупреждения

## База данных

### chats
Чаты под защитой с настройками (captcha_enabled, scoring_enabled, scoring_threshold, welcome_message, rules_message и т.д.)

### join_events
Лог всех вступлений для подсчёта в скользящем окне

### attack_sessions
История атак со статистикой

### pending_captcha
Кто сейчас проходит капчу: chat_id, user_id, правильный ответ, время истечения

### good_users
Пользователи, прошедшие проверку (капча/скоринг) - используется для статистики скоринга за последние 7 дней

### scoring_adjustments
История автокорректировок скоринга: что изменилось, когда и на каких данных

### app_settings
Глобальные настройки бота (включая ежедневный дайджест и время рассылки)

### stop_words
Нормализованные стоп-слова на чат (в нижнем регистре)
## Логи

Логи сохраняются в `logs/[username или chat_id]/YYYY-MM-DD.log`

Формат:
```
[2025-12-08 14:30:15] INFO: JOIN | USER | ID: 123456789 | Username: john_doe
[2025-12-08 14:30:45] WARNING: ATTACK STARTED | Threshold: 10 | Detected: 12 joins
[2025-12-08 14:30:46] INFO: KICK | ID: 987654321 | Username: bot_user | Reason: protection
[2025-12-08 14:35:20] WARNING: ATTACK ENDED | Duration: 4m 35s | Total kicked: 87
```
